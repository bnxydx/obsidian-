# 我的笔记风格分析

> 由 Marvis 分析 C:\D\obsidian笔记\408\操作系统 目录后总结，2026-06-09

---

## 组织结构

- **目录层级**：学科根目录 → 章目录 → (功能子目录)，2~3 层
- **文件命名**：`节号+知识点名.md`，纯中文，重要文件加 `（重要）`
- **索引体系**：三级 MOC（根→章→知识点），`[[wikilink]]` 互联
- **图片管理**：统一放 `picture/` 子目录，Obsidian 默认粘贴名

---

## 内容格式

| 维度 | 习惯 |
|---|---|
| 标题 | H1 / H2 为主，**加粗** + 缩进列表替代深层标题 |
| Tag | 不使用 `#tag` 和 YAML frontmatter |
| 链接 | `[[wikilink]]` 为主，与 `[text](file.md)` 混用 |
| Callout | `>` 引用块，**不用** `[!note]` 语法 |
| 分隔线 | `---` 隔离主题块 |
| 表格 | 高频，对比类知识点必用，表头加粗、内容极简 |
| 代码块 | 仅 `txt` 和 `mermaid`，mermaid 带 style |
| 公式 | `$行内$` 为主，`\text{}` 包裹中文 |

---

## 书写风格

- **加粗强调关键词/考点**，如 **并发**、**系统调用**
- 口语化但专业，"一句话"总结法
- 考试导向批注："后面二轮看"、"408 常见连线/考点"
- 中等偏详细：定义 → 特征列表 → 对比表格 → 考点总结
- 核心结论用 `>` 引用块收尾

---

## 隐式创作模板

```
# 知识点名称
  ↓
【定义】加粗一句话定义
  ↓
【分点展开】缩进列表 / 特征列表
  ↓
【对比】表格
  ↓
【结论/考点】引用块总结
```

## 其他习惯

- 每章独立 `错题/` 子目录，错题用截图 + 知识点联动
- 跨学科用 wikilink 指向其他目录
- `0.0跳过的课.md` 记录未学内容和二轮计划
- 内部锚点链接精确到小节
