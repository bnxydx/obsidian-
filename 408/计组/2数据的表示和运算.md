## [[数据的宽度和单位]]

- **比特（bit，也称位，符号 b）**：最小的信息单位，表示一个二进制位（0 或 1）。
- **字节（byte，符号 B）**：基本的存储和寻址单位，1 字节 = 8 比特。容量前缀按 2 的幂定义，如 **1KB = 2¹⁰B = 1024B**。
- **字（word）**：由体系结构定义的逻辑单位，用于表示整数、地址等基本数据类型的宽度，长度因架构而异，常见 2、4 或 8 字节。
- **字长（机器字长）**：CPU 内部整数运算数据通路的宽度，通常等于通用寄存器宽度，反映计算机一次能处理的整数位数。日常所说的"32 位机"或"64 位机"即指字长。

> 字是架构层面的约定，而字长体现的是硬件的实际处理能力。例如 Intel x86 架构中自 80386 起字长为 32 位，但其体系结构仍将 16 位定义为一个字，32 位称为双字。

## [[二、 进制转换核心关系]]

在计算机基础中，各进制间常见的转换路径如下：

1.  **十进制 $\rightleftarrows$ 二进制**
2.  **二进制 $\rightleftarrows$ 十六进制**
3.  **十进制 $\rightleftarrows$ 十六进制** （通常可以通过二进制作为中介快速转换，也可以直接计算）

## [[三、 不同进制的按权展开式 (重点)]]

通过将各进制数按权展开，可以直观地将其转换为**十进制数**。

### 1. 整数部分的按权展开

-   **十进制**：$1234 = 1 \times 10^3 + 2 \times 10^2 + 3 \times 10^1 + 4 \times 10^0$
-   **二进制**：$1011 = 1 \times 2^3 + 0 \times 2^2 + 1 \times 2^1 + 1 \times 2^0$
-   **十六进制**：$BF6D = 11 \times 16^3 + 15 \times 16^2 + 6 \times 16^1 + 13 \times 16^0$ （注：$B=11, F=15, D=13$）
-   **八进制**：$6724 = 6 \times 8^3 + 7 \times 8^2 + 2 \times 8^1 + 4 \times 8^0$

### 2. 含有小数部分的按权展开

当数字包含小数时，小数点右侧的数位权值依次为底数的负整数次幂（$-1, -2, -3 \dots$）：

-   **十进制小数**：

    $$123.456 = 1 \times 10^2 + 2 \times 10^1 + 3 \times 10^0 + 4 \times 10^{-1} + 5 \times 10^{-2} + 6 \times 10^{-3}$$

-   **二进制小数**：

    $$101.101 = 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0 + 1 \times 2^{-1} + 0 \times 2^{-2} + 1 \times 2^{-3}$$

移位：小数点固定，把数整体左移，右移

~~~

~~~

### 1. 左移 2 位（未溢出）

-   **过程：** `0034` $\xrightarrow{\text{左移 2 位}}$ `3400`
-   **数学关系：** $3400 = 34 \times 10^2$

### 2. 左移 2 位（溢出）

-   **过程：** `1234` $\xrightarrow{\text{左移 2 位}}$ `3400`
-   **数学关系：** $3400 \neq 1234 \times 10^2$ ，**溢出**

### 3. 右移 2 位（未丢失精度）

-   **过程：** `3400` $\xrightarrow{\text{右移 2 位}}$ `0034`
-   **数学关系：** $0034 = 3400 / 10^2$

### 4. 右移 2 位（丢失精度）

-   **过程：** `1234` $\xrightarrow{\text{右移 2 位}}$ `0012`

-   **数学关系：** $0012 \neq 1234 / 10^2$ ，**丢失精度**

    *(注：实际值为 12.34)*

### 逻辑移位（无符号数）

### 1. 左移 2 位

-   `0011 (3)` $\xrightarrow{\text{左移 2 位}}$ `1100 (12) = 3 × 4`

### 2. 右移 2 位

-   `1100 (12)` $\xrightarrow{\text{右移 2 位}}$ `0011 (3) = 12 / 4`
-   `1011 (11)` $\xrightarrow{\text{右移 2 位}}$ `0010 (2) ≠ 11 / 4`（丢失精度）

## IEEE754

现代系统普遍采用 IEEE 754 浮点数标准。该标准定义了两种常用格式：**32 位单精度浮点数（float 型）** 和 **64 位双精度浮点数（double 型）**，其基数隐含为 2，其格式如下。

**32 位单精度格式：**

| 符号 | 阶码 | 尾数 |
|:---:|:---:|:---:|
| 1 位 | 8 位 | 23 位 |
1. **阶码使用移码的方式存且移码**为$$2^{N-1}-1 = 2^7-1=127$$
**64 位双精度格式：**

| 符号 | 阶码 | 尾数 |
|:---:|:---:|:---:|
| 1 位 | 11 位 | 52 位 |
阶码使用移码的方式存
且移码为$$2^{N-1}-1 = 2^10-1=1023$$

2. 最后还需要转成16进制
3. 尾数的1.不用存(隐藏位)

## 浮点数的表示范围

IEEE 754 标准为了处理一些特殊情况，硬性剥夺（保留）了两个极端边界：

- **保留 `0000 0000`（全0）：** 用于表示“机器数0”或“非规格化数”。
    
- **保留 `1111 1111`（全1）：** 用于表示“无穷大（$\pm\infty$）”或“非数（NaN）”。
    

由于**全1被保留用来表示无穷大**了，它就不能再用来表示正常的规格化浮点数。

### 浮点数特殊值的表示

1. **全 0 阶码全 0 尾数**：+0 / -0。符号 s 决定其正负，通常情况下 +0 和 -0 是等效的。

2. **全 1 阶码全 0 尾数**：+∞ / -∞。+∞ 在数值上大于所有有限数，-∞ 则小于所有有限数。引入无穷大的目的是使程序在溢出等异常情况下仍能继续执行。

3. **全 1 阶码非 0 尾数**：NaN（Not a Number）。表示一个没有定义的数，称为非数。

4. **全 0 阶码非 0 尾数**：非规格化数。非规格化数的特点是阶码为全 0，不使用隐藏位，尾数字段不全为 0。单精度和双精度浮点数的指数分别为 -126 和 -1022。非规格化数用于实现渐进下溢，填补 0 与最小规格化数之间的数值间隙。

# 数据大小端存放

在多字节数据（如 `int` 型占 4 字节）存入内存时，由于内存地址是按字节（Byte）编址的，这就涉及到多字节在连续内存地址中的**字节排列顺序**问题。

数据准备：以 32 位十六进制数 `0x01234567H` 为例，从高位到低位依次为：
* **高字节 (MSB)：** `01H`
* **次高字节：** `23H`
* **次低字节：** `45H`
* **低字节 (LSB)：** `67H`

---
### 1. 大端方式 (Big-Endian)
* **定义：** 数据的**高字节**存放在内存的**低地址**端，数据的低字节存放在内存的高地址端。
* **特点：** 看起来最符合人类从左到右的阅读习惯。
* **内存映射 ($0800\text{H} \sim 0803\text{H}$)：**

| 内存地址 | 数据字节 | 字节属性 |
| :--- | :--- | :--- |
| **0800H (低地址)** | `01H` | 高字节 |
| **0801H** | `23H` | 次高字节 |
| **0802H** | `45H` | 次低字节 |
| **0803H (高地址)** | `67H` | 低字节 |

### 2. 小端方式 (Little-Endian)
* **定义：** 数据的**低字节**存放在内存的**低地址**端，数据的高字节存放在内存的高地址端。
* **特点：** x86、ARM 等现代 CPU 常用的底层存储方式。
* **内存映射 ($0800\text{H} \sim 0803\text{H}$)：**

| 内存地址 | 数据字节 | 字节属性 |
| :--- | :--- | :--- |
| **0800H (低地址)** | `67H` | 低字节 |
| **0801H** | `45H` | 次低字节 |
| **0802H** | `23H` | 次高字节 |
| **0803H (高地址)** | `01H` | 高字节 |

---

```text
 低地址  0800H  [  01H  ]  (高字节)
         0801H  [  23H  ]
         0802H  [  45H  ]
 高地址  0803H  [  67H  ]  (低字节)

```

# 边界对齐

在计算机中，为了提高 CPU 访问内存的效率，结构体成员在内存中的起始地址不能随便排布，必须遵循“边界对齐规则”**。

---

## 一、 核心对齐两大规则

假设结构体起始总地址为 `0`：

1. **成员对齐规则：** 每个成员的**起始偏移地址**，必须是**该成员自身大小（对齐值）**的整数倍。
   * `char` (1B)：起始地址可以是 0, 1, 2, 3...（任意地址）
   * `short` (2B)：起始地址必须是 2 的倍数，如 0, 2, 4, 6, 8...
   * `int` (4B)：起始地址必须是 4 的倍数，如 0, 4, 8, 12...
2. **结构体整体对齐规则：** 结构体的**总大小（sizeof）**，必须是**结构体中最大成员大小**的整数倍。如果不足，必须在结构体末尾填充空字节（Padding）。

---

```
struct A{
	int a;
	char b;
	short c;
}
```
```
int a起始时0，符合条件一是int(4)的整数倍
char b 起始时4 符号条件一是char(1)的整数倍
short c 起始5，不符合条件1，加一个=6，是short(2)的整数倍

最终大小为8是最大成员int(4)的整数倍，不需要补
```
