# [[机器指令的结构]]



操作码 + 寻址特征+形式地址

### 1. 操作码（OP, Operation Code）

-   **它是干啥的**：**告诉 CPU 这条指令要做什么操作（指明操作的性质）。**

### 2. 寻址特征（Addressing Mode Indicator）

-   **它是干啥的**：**告诉 CPU 应该用什么“方法”去寻找真正的数据。**

寻址特征 = 立即寻址

**操作码**：`ADD`（加法）

**形式地址 $A$**：`100`

**CPU 的动作**：寻址特征说“这是立即数”，所以 $A$ 就是数据本身！CPU 直接把数字 `100` 加到累加器里。

### 3. 形式地址（A, Formal Address）

-   **它是干啥的**：**指令里表面上写着的地址（或数值），但它通常不是数据最终在内存中的绝对位置。**

为什么非要加一个“寻址特征”，而不是直接把最终地址写死在指令里？

1.  **为了扩大寻址范围**：指令的长度是有限的（比如16位），分给地址码的可能只有 6 位（只能找 64 个单元）。但如果配合寻址特征使用“间接寻址”或“寄存器寻址”，就能利用寄存器或者内存单元的整段位数（如32位），去访问极大的内存空间。
2.  **为了方便程序编写**：有了寻址特征，我们才能支持数组访问（变址寻址）、多道程序在内存里动态移动（基址寻址）以及函数调用等高级操作。
