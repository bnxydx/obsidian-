# 同步和互斥的基本概念

## 同步
**临界资源**：同一时刻仅允许一个进程访问的资源
既有硬件也有软件
例：打印机，共享变量，数据结构

**对临界资源必须要互斥访问**
**临界区**：访问临界资源的代码，不能用多个进程同时执行这段代码
有以下四个部分
1. 进入区：检查当前能不能进，进入后防止其他进程进入
2. 临界区：访问临界资源的代码段
3. 退出区：释放
4. 剩余区

**同步**
协调进程的运行次序，再执行过程产生等待或传递信息的制约关系

**互斥**
当一个进程正在临界区使用临界资源的时候，其他访问改资源的进程需等待，只有没有人用的时候才可以进入 

**同步机制：**

**空闲让进
忙则等待
有限等待
让权等待**

## 信号量

用来表示某种资源的数量，一个打印机S = 1
信号量需要使用两个标准的原语（不可被中断）
`wait(),signal()`
`P(S)操作  ，V(S)操作`

1. 整形信号量
用一个正数表示某种资源
init
P
V
```C
int S = 1;  // 初始化整型信号量s，表示当前系统中可用的打印机资源数

void wait (int S) {      //wait 原语，相当于“进入区”
    while (S <= 0);       //如果资源数不够，就一直循环等待
    S=S-1;               //如果资源数够，则占用一个资源
}

void signal (int S) {    //signal 原语，相当于“退出区”
    S=S+1;               //使用完资源后，在退出区释放资源
}

```

```txt
进程P0:

wait(S);               //进入区，申请资源
使用打印机资源...        //临界区，访问资源
signal(S);               //退出区，释放资源

进程P1:

wait(S);
使用打印机资源...
signal(S);

进程Pn:

wait(S);
使用打印机资源...
signal(S);
```
这里会一直不会被切换
**会发生忙等**
不满足让权等待
1. 记录型信号量
```C
/*记录型信号量的定义*/
typedef struct {
    int value;          //剩余资源数
    struct process *L;  //等待队列
} semaphore;

/*某进程需要使用资源时，通过 wait 原语申请*/
void wait (semaphore S) {
    S.value--;
    if (S.value < 0) {
        block (S.L);
    }
}

/*进程使用完资源后，通过 signal 原语释放*/
void signal (semaphore S) {
    s.value++;
    if (S.value <= 0) {
        wakeup(S.L);
    }
}

```
