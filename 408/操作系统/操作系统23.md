## 第三章处理机

~~~
处理机调度 (Processor Scheduling)
│
├── 一、调度层次（三级调度模型）
│   │
│   ├── 1.1 作业调度（高级调度 / 长程调度）
│   │   ├── 作用：决定外存作业 → 调入内存 → 创建进程
│   │   ├── 对象：作业(Job)
│   │   ├── 频率：低（分钟/小时级）
│   │   └── 目标：控制并发度、平衡资源
│   │
│   ├── 1.2 中级调度（内存调度 / 中程调度）
│   │   ├── 作用：挂起/激活进程，换入换出内存 ↔ 外存
│   │   ├── 对象：进程(Process)
│   │   └── 目标：提高内存利用率、系统吞吐量（虚拟内存管理核心）
│   │
│   └── 1.3 进程调度（低级调度 / 短程调度 / CPU调度）
│       ├── 作用：从就绪队列选进程 → 分配CPU
│       ├── 对象：进程/线程
│       ├── 频率：高（毫秒级）
│       └── 目标：CPU利用率、响应时间、公平性、吞吐量
│
├── 二、调度目标
│   ├── CPU利用率最大化
│   ├── 吞吐量最大化（单位时间完成进程数）
│   ├── 周转时间最小化（提交→完成）
│   ├── 等待时间最小化（在就绪队列总时间）
│   ├── 响应时间最小化（请求→首次响应）→ 交互式系统关键
│   └── 公平性（避免饥饿，所有进程合理获得服务）
│
├── 三、调度算法分类（按是否抢占）
│   │
│   ├── 3.1 非抢占式调度 (Non-preemptive)
│   │   ├── 一旦运行，除非主动放弃CPU，否则不被打断
│   │   ├── 优点：实现简单，上下文切换少
│   │   └── 缺点：响应慢，不适合交互系统
│   │
│   └── 3.2 抢占式调度 (Preemptive)
│       ├── 可被系统强制中断，CPU可被更高优先级进程抢占
│       ├── 优点：响应快，实时性好
│       └── 缺点：实现复杂，需处理同步/一致性
│
├── 四、经典调度算法详解
│   │
│   ├── 4.1 先来先服务 FCFS (First-Come, First-Served)
│   │   ├── 类型：非抢占
│   │   ├── 特点：按到达顺序执行
│   │   ├── 优点：公平、简单
│   │   └── 缺点：护航效应，平均等待时间长，不利于短作业
│   │
│   ├── 4.2 短作业/进程优先 SJF/SPF (Shortest Job/Process First)
│   │   ├── 类型：可抢占(SRTN) / 非抢占
│   │   ├── 特点：优先执行预计运行时间最短者
│   │   ├── 优点：平均等待时间最小（理论最优）
│   │   └── 缺点：需预知运行时间；长作业可能饥饿
│   │
│   ├── 4.3 时间片轮转 RR (Round Robin)
│   │   ├── 类型：抢占式
│   │   ├── 特点：固定时间片，循环执行
│   │   ├── 优点：公平、响应快、无饥饿
│   │   └── 缺点：时间片选择关键（太小→切换开销大；太大→退化FCFS）
│   │
│   ├── 4.4 优先级调度 (Priority Scheduling)
│   │   ├── 类型：可抢占 / 非抢占
│   │   ├── 特点：按优先级高低调度
│   │   ├── 优点：灵活，可区分任务重要性
│   │   └── 缺点：低优先级进程可能饥饿 → 解决：老化(Aging)
│   │
│   ├── 4.5 最高响应比优先 HRRN (Highest Response Ratio Next)
│   │   ├── 类型：非抢占（通常用于作业调度）
│   │   ├── 公式：响应比 = (等待时间 + 服务时间) / 服务时间
│   │   ├── 优点：兼顾等待时间和运行时间，防饥饿
│   │   └── 缺点：需预知服务时间，计算开销
│   │
│   └── 4.6 多级反馈队列 MFQ (Multilevel Feedback Queue)
│       ├── 类型：抢占式
│       ├── 特点：
│       │   - 多个优先级队列（Q0最高，Qn最低）
│       │   - 新进程→Q0，时间片小（如8ms）
│       │   - 未完成→降级到下一队列，时间片加倍（如16ms, 32ms...）
│       │   - 老化机制：长时间等待→提升优先级
│       ├── 优点：兼顾响应（短/I/O进程快）与吞吐（长进程最终执行），防饥饿
│       └── 应用：Unix、Windows、Linux等通用OS核心调度算法
│
├── 五、调度算法选择依据
│   ├── 系统类型：
│   │   ├── 批处理系统 → 吞吐量、周转时间 → SJF、HRRN
│   │   ├── 分时系统 → 响应时间、公平 → RR、MFQ
│   │   └── 实时系统 → 截止时间保障 → 专用实时调度算法（EDF, RMS）
│   │
│   ├── 进程特征：
│   │   ├── CPU密集型 vs I/O密集型
│   │   └── 长进程 vs 短进程
│   │
│   └── 性能指标权衡：
│       ├── 响应性 vs 吞吐量
│       └── 公平性 vs 效率
│
└── 六、现代操作系统实践
    ├── Linux：CFS (Completely Fair Scheduler) —— 基于红黑树的公平调度
    ├── Windows：优先级+多级反馈队列+时间片+抢占
    └── 实时系统：Rate-Monotonic (RMS), Earliest Deadline First (EDF)
~~~
