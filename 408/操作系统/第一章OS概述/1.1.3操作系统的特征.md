## 并发&并行

**并发**：多个程序在**同一时间间隔**内交替执行，宏观同时、微观轮流，单核 CPU 即可实现。

**并行**：多个程序在**同一时刻**真正同时执行，必须靠多核或多处理器。
CPU1
CPU2...同一时刻一起执行

> 单核上的多道程序是并发，不是并行。

而CPU与I/O 设备之间、多个IⅣO设备之间则可实现**真正的硬件并行**。
若要实现**进程**级别的**并行**，则需多核CPU或多处理机等硬件支持。

## 共享
互斥共享方法
1. 一个资源虽然可以让多个进程使用，但是一次只能给一个进程用
2. 这个资源叫做临界资源
3. 例如：打印机，磁带机

同时访问方式
1. 并发访问
2. 分时交替
3. 访问磁盘文件

---
并发和共享是最基本的特征

## 虚拟
**操作系统虚拟化**：
实现的两种技术：

- **时分复用** → CPU 虚拟化：通过多道程序+时间片轮转，让多程序并发，用户感觉独占 CPU
- **空分复用** → 内存虚拟化：通过虚拟存储器，将有限物理内存扩展为更大逻辑地址空间

- **设备虚拟化**：如 SPOOLing 技术，把独占设备（打印机）变成逻辑共享设备

本质：将物理实体映射为逻辑抽象，让用户面对的是虚拟资源而非真实硬件。

## 异步
在多道程序环境下，多个进程并发执行。但由于系统资源有限，进程的执行并非连续进行，而是以**不可预知**的速度断续推进，这种特性称为进程的异步性。
```txt
P1   P2
1    1
2    2
3    3

随机切换的
```
