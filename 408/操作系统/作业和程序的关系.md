**程序**是静态的指令序列，存储在磁盘上。

**作业**是用户提交给系统的一项完整任务，包含**程序 + 数据 + 作业说明书**。

关系：作业比程序范围大。一个作业可以包含多个程序，比如编译作业需要先运行编译程序、再运行链接程序。早期批处理系统中以作业为调度单位，现在多道程序系统中以进程为单位。


# 举例
以**银行批量处理年终报表**为例：

用户提交一份作业，包含三部分：

| 组成部分  | 内容                        | 形式            |
| ----- | ------------------------- | ------------- |
| 程序 1  | 从数据库提取全年交易数据              | `extract.exe` |
| 程序 2  | 按分行分类统计并生成报表              | `report.exe`  |
| 程序 3  | 压缩打包发送到各分行邮箱              | `send.exe`    |
| 数据    | 数据库路径、各分行邮箱列表             | 参数文件          |
| 作业说明书 | 三个步骤的执行顺序、失败重试策略、CPU/内存需求 | JCL（作业控制语言）   |

**区别一目了然**：

> 程序 = 单独的 `.exe` 文件，谁都可以拿来用。
> 
> 作业 = 这次银行报表任务的整体，指定了「用哪个程序 → 处理哪些数据 → 出错了怎么办」。

再比如编程课上交的 C 语言大作业：
- 你写的 `main.c` 是**程序**
- 挂在在线评测系统上跑的那次提交（包含代码 + 测试数据 + 时间限制 + 内存限制）是一个**作业**

**一句话**：程序是工具，作业是用工具完成一次具体任务。
