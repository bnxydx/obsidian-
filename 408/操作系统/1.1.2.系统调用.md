# 系统调用

## 一、定义

系统调用（System Call）是操作系统提供给应用程序的**程序接口**，是应用程序**访问系统资源、请求操作系统服务**的**唯一合法途径**。

系统中各种资源必须由操作系统统一管理，都需要通过系统调用向操作系统提出服务请求，并让操作系统完成，把结果返回给应用程序

能够让程序调用内核的功能

一般是主动进入内核
## 二、在操作系统接口体系中的位置

操作系统提供两类接口：

| 接口类型 | 面向对象 | 示例 |
|---------|---------|------|
| 命令接口（用户接口） | 普通用户 | CLI（命令行）、GUI（图形界面）、批处理 |
| **程序接口（系统调用）** | 编程人员 / 应用程序 | `write()`、`open()`、`fork()` |

> 操作系统接口主要有命令接口和程序接口，程序接口也称系统调用。

## 三、系统调用 vs 库函数

| | 系统调用 | 库函数 |
|---|---------|-------|
| 运行状态 | 内核态 | 用户态（大多数） |
| 所属 | 操作系统内核 | 用户程序 |
| 开销 | 大（需切换内核态） | 小 |
| 本质 | 操作系统真正的接口 | 对系统调用的封装 |

- 库函数（如 `printf()`、`malloc()`）是对系统调用的封装，本身不是操作系统接口
- `printf()` 最终通过 `write` 系统调用输出数据
- `malloc()` 可能调用 `brk` 或 `mmap` 系统调用
- **纯计算类库函数**（如 `strlen()`、`sqrt()`）在用户空间运行，不涉及系统调用
- 输入输出都是内核态

## 四、系统调用与一般函数调用的区别

|      | 系统调用           | 一般函数调用 |
| ---- | -------------- | ------ |
| 运行级别 | 从用户态切换到内核态     | 同一特权级内 |
| 入口地址 | 内核维护（中断/陷入向量表） | 用户程序内  |
| 开销   | 大（特权级切换、上下文保存） | 小      |
| 安全性  | 受控访问           | 无      |
|      |                |        |
**系统调用在用户态，执行在内核态**
## 五、历史演变

- 早期系统调用基于**汇编语言**实现，仅在汇编程序中可直接使用
- 高级语言（如C）中，程序员通过**标准库函数**间接触发系统调用

## 六、常见系统调用分类

| 类别   | 功能        | 示例                                    |
| ---- | --------- | ------------------------------------- |
| 进程控制 | 创建、终止、等待  | `fork()`、`exit()`、`wait()`            |
| 文件管理 | 打开、读写、关闭  | `open()`、`read()`、`write()`、`close()` |
| 设备管理 | 请求/释放设备   | `ioctl()`                             |
| 通信   | 进程间通信     | `pipe()`、`shmget()`                   |
| 信息维护 | 获取系统时间/信息 | `getpid()`、`time()`                   |
|      |           |                                       |

## 七、系统调用执行过程

系统调用的处理过程可分为 **4 个阶段**：

1. **传递系统调用参数** — 用户程序将系统调用号和参数存入约定寄存器或栈中
2. **执行陷入指令并切换至内核态** — 触发 trap（访管） 指令，CPU 从用户态切换到内核态，保存现场
3. **定位并执行相应服务程序** — 内核根据调用号查系统调用表，跳转到对应服务例程执行
4. **恢复现场并返回用户态** — 执行完毕，恢复之前保存的上下文，返回用户程序继续执行

## 八、关键考点

1. 系统调用是程序接口，不是命令接口
2. 库函数属于用户程序，不是系统调用
3. 系统调用的目的是请求系统服务
4. 单核上的多道程序是并发，不是并行
5. 纯计算类库函数不涉及系统调用（并非所有库函数都需要系统调用）
6. 执行系统调用的时候是处于内核态
