# 分页管理
**离散存储**
无外部碎片，可能内部碎片(最后一页可能没被填满)
## 基本概念

将进程的**逻辑地址空间**划分为若干固定大小的块，称为**页面(页)（Page）**；将**内存物理地址空间**划分为同样大小的块，称为**页框/物理块（Frame）/页帧**。

> 页面大小 $=$ 页框大小，通常为 $2$ 的幂次（如 $4\text{KB}$）。分配时以页框为单位，**页内不产生外部碎片**。

---

## 页面与页框

| 概念 | 所属空间 | 说明 |
|------|---------|------|
| **页面（Page）** | 逻辑地址空间 | 进程按固定大小切分后的块，编号为页号 $0,1,2,\dots$ |
| **页框（Frame）** | 物理地址空间 | 内存按同样大小切分后的块，编号为块号 $0,1,2,\dots$ |

- 页面与页框**大小严格相等**，一页恰好装入一框
- 进程的页面可**离散分配**到不连续的物理页框中
- 最后一页可能有**内部碎片**（页内未用完的空间）

---

## 页表项（PTE）结构

每个页面在**页表**中对应一条**页表项（Page Table Entry）**，核心字段：

、

| 字段 | 作用 |
|------|------|
| **页框号（物理块号）** | 该页映射到的物理内存块编号 |
| **有效/无效位**（Present） | 该页是否已在内存中；无效时访问触发**缺页中断** |
| **访问位**（Reference） | 记录该页是否被访问过，供**页面置换算法**（如 NRU/Clock）决策 |
| **修改位**（Dirty） | 该页内容是否被修改过；换出时决定是否需要**写回磁盘** |
| **保护位**（Protection） | 权限控制（读 $R$ / 写 $W$ / 执行 $X$） |

> 有效位 $=0$ 表示缺页，修改位 $=1$ 表示脏页需写回。两者是考研高频考点。

---

## 逻辑地址结构

**逻辑地址** $A$ 分为两部分：

- **页号** $P$：对应页表中的索引
- **页内偏移量** $W$（也称页内地址）
![](Pasted%20image%2020260612192815.png)
根据页面大小确定

> 计算公式：$P = \lfloor A / L \rfloor$，$W = A \bmod L$，其中 $L$ 为**页面大小**。

**示例**：逻辑地址 $11$，页面大小 $4$ → $11 \div 4 = 2 \dots 3$，即页号 $2$，页内偏移 $3$。

---

## 地址映射过程

逻辑地址 $\to$ 物理地址的转换过程：

1. 从逻辑地址中拆分出 **页号** 和 **页内偏移**
2. 以页号为索引查 **页表**，获取对应 **物理块号**（帧号）
3. **页内偏移直接拼接**到物理块号后面，得到最终 **物理地址**

> 页内偏移量在映射过程中**保持不变**，仅页号被替换为物理块号。

## 完整计算示例

> 已知：**页面大小** $L = 1\text{KB} = 1024\text{B}$，逻辑地址 $A = 2500$，页表：页号 $0 \to$ 块号 $5$，页号 $1 \to$ 块号 $2$，页号 $2 \to$ 块号 $7$。求物理地址。

**Step1--判断是否越界，会触发越界中断，否则继续**
**Step 2 — 拆分逻辑地址：**

$$P = \lfloor 2500 / 1024 \rfloor = 2 \quad (\text{页号})$$

$$W = 2500 \bmod 1024 = 452 \quad (\text{页内偏移})$$

**Step 3 — 查页表：**

页号 $2$ 对应物理块号 $b = 7$。

**Step 4 — 拼接物理地址：**

$$\text{物理地址} = b \times L + W = 7 \times 1024 + 452 = 7620$$

> 逻辑地址 $2500$ $\to$ 物理地址 $7620$。页内偏移 $452$ 全程不变，仅页号由 $2$ 替换为块号 $7$。

## 二级页表

### 引入原因

单级页表需**连续存放**，大地址空间下页表本身过于庞大。如 $32$ 位系统，页面 $4\text{KB}$，页表项 $2^{20} = 1\text{M}$ 条，每条 $4\text{B}$，总大小 $4\text{MB}$，必须连续。

> 将长页表分组离散存放，只调入正在使用的部分，即为**多级页表**的思路。

### 两级页表结构
高五位是一级页表的序号，中五位是二级页表的偏移，最后是物理地址的偏移

| 层级                  | 名称   | 作用                |
| ------------------- | ---- | ----------------- |
| **外层页表**（页目录）(页目录表) | 顶级页表 | 每个表项指向一个内层页表的物理块号 |
| **内层页表**            | 二级页表 | 每个表项指向最终的数据页框号    |

逻辑地址结构：$(\text{外层页号 } P_1,\ \text{内层页号 } P_2,\ \text{页内偏移 } W)$

## 地址转换流程

1. 从**页目录基址寄存器**取出外层页表地址
2. $P_1$ 查外层页表 → 获取内层页表所在物理块号
3. $P_2$ 查内层页表 → 获取目标数据页框号
4. 页框号 $+W$ → **物理地址**

> 访存次数：单级 $2$ 次，**二级 $3$ 次**（外层表 $+$ 内层表 $+$ 数据）。需配合 **TLB 快表**降低访存开销，是核心考点。

## 核心优点

- **离散存放**：页表各页可分散到不同物理块，无需连续 $4\text{MB}$ 空间
- **按需调入**：未使用的页表项对应的内层页表可不创建/不调入，大大节省内存
- **可扩展**：同样思路推广至多级（$64$ 位系统常用 $4$ 级页表）

# 分页与分段的对比

分页和分段均为**非连续分配**方式，均需通过**地址映射机构**实现地址变换。但二者在概念上有本质区别：

| 对比维度     | **分页**                       | **分段**                                    |
| -------- | ---------------------------- | ----------------------------------------- |
| **划分单位** | 物理单位（系统管理内存）                 | 逻辑单位（用户程序结构）                              |
| **目的**   | 提高**内存利用率**                  | 满足**编程、共享、保护**需求                          |
| **实现方**  | 系统实现，**对用户透明**               | 编译器划分，**对用户可见**                           |
| **大小**   | **固定**，由系统决定（如 $4\text{KB}$） | **不固定**，取决于程序逻辑结构                         |
| **地址空间** | **一维**：单一线性地址                | **二维**：必须同时指定 $(\text{段号},\ \text{段内偏移})$ |

> 分页是为了"把内存用满"，分段是为了"把程序写好"。一维 vs 二维地址是选择题高频考点。
