{
	if(T == NULL)
		return NULL;
	if(key == T->key)
		return T;
	else if(T -> key > key)
		return BST_Search(T -> lchild,key);
	else BST_Search(T -> rchild,key);
}

## 插入

~~~
int BST_Insert(BSTree &T,int k)
{
	if(T == NULL){// 新插入为根
		T = (BSTree)malloc(sizeof(BSTNode));
		T -> key = k;
		T -> lchild = T -> rchild = NULL;
		return 1;
	}
	else if(k == T -> key)
		return 0; // 相同的数字
	else if(k > T -> key)
		return BST_Insert(T -> lchild,k); // 插入到左子树
	else:
		return BST_Insert(T -> rchild,k)
		// 插入到右子树
}
~~~

# B树

## 王道

![image-20260425091705991](./picture/image-20260425091705991.png)

多路平衡查找树

B树的阶：m：所有节点的孩子的个数的最大值

-   树中每个节点最多m棵子树，至多有m-1个关键字
-   若根节点不是终端结点，至少有两棵子树
-   出根节点外，所有非叶节点至少有**(m+1)/2**棵子树，即至少有**(m+1)/2-1**个关键字
-   所有叶节点都在**同一层上**，并且是NULL
-   所有子树的**同高度的**
-   所有的关键字都是树内结点，并且代表下一层的区间，例如关键字是a b c，对应下一层就是<a,(a,b),(b,c),c<

-   树内节点都是递增的

$log_m(n+1)<=h<=log_{(m+1)/2}(n+1)/2+1$

-   **最小高度**（不包括叶节点），让每个结点满，n<=(1 + m,m*m,...+m^h-1)
    -   $$h >= log_m(n+1)$$

-   **最大高度**,根节点最少有两个分叉，其他结点只有(m+1)/2个分叉

![image-20260425092806687](./picture/image-20260425092806687.png)

![image-20260425092517188-1777080318458-1](./picture/image-20260425092517188-1777080318458-1.png)

-   n个关键字的B树必有n+1个叶子节点

## 抖音

**(a,b)-tree**

-   节点里存放键值，并且递增
-   叶子节点不放任何值
-   每个键值都分隔两个子树
    -   左边放小于自己的值
    -   右边..
-   每个结点存放的数量由ab决定

-   a是每个结点所拥有的最小的子节点数
    -   必须根节点个数=1，那么拥有两个子树，就是两个子节点数
-   b是每个结点最大的子节点数

**其他结点所包含的结点数必须在a，b之间**

**根节点的节点数必须在2,b之间**

-   叶子节点在同一层
-   **a>=2,b>=2a-1**

m阶

a = (m+1)/2 - 1

b = m - 1
