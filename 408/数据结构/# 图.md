## dijkstra O（N*N）

给定一个 n 个点 m 条边的有向图，图中可能存在重边和自环，所有边权均为正值。

请你求出 1 号点到 n 号点的最短距离，如果无法从 1 号点走到 n 号点，则输出 −1。

-   负权图不能用

~~~
dist[n] //用于存储每个点到起点的最短距离
st[n]   //用于在更新最短距离时 判断当前的点的最短距离是否确定 是否需要更新
~~~



-   初始化距离 dist[1] = 0   1号点到的距离为0，dist[i] = 0x3f
-   s 当前已经确定了最短距离的点
-   for i in range(n):
    -   找到当前不在s中的距离最短的点
    -   t 加入s
    -   t 更新其他所有点的距离（从t出去所有的边，组成的路径能不能更新其他点的距离）

-   初始化
-   选最小的点加入集合
-   更新

## Floyd

**不允许有负权回路**

给定一个 n 个点 m 条边的有向图，图中可能存在重边和自环，边权可能为负数。

再给定 k 个询问，每个询问包含两个整数 x 和 y，表示查询从点 x 到点 y 的最短距离，如果路径不存在，则输出 `impossible`。

-   求多源汇最短路

~~~
d[i][j] 存所有的边
d[k][i][j] 从i点只经过k点到达j点的最短距离
d[k][i][j] = d[k - 1][i][k] + d[k - 1][k][j] 
三维优化



for(int k = 1; k < n;k ++)	
	for(int i = 1 ;i <= n ;i ++)
		for(int j = 1 ;j <= n ; j ++)
			d[i][j] = min(d[i][j],d[i][k] + d[k][j])
			
			
d[i][j] 最短路的长度
~~~

## 强连通分量

### 1. 最直观的定义：相互通达的“小圈子”

在一个有向图中，如果两个点 $A$ 和 $B$ 能够“**你中有我，我中有你**”——即从 $A$ 出发能走到 $B$，同时从 $B$ 出发也能走回 $A$，那么这两个点就是“强连通”的。

**强连通分量**就是图中最大的这种“互通小圈子”。在这个圈子里：

-   **任意两个点**之间都可以通过有向路径互相到达。
-   **最大性**：如果你尝试把圈子外的任何一个点拉进来，这个“互相到达”的规则就会被打破。
