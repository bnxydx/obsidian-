# 使用先验锚框

在YOLOv1 ~ YOLOv7几乎都在使用先验锚框

| 版本                        | 是否使用 Anchor  | 关键变化与原因                                               |
| --------------------------- | ---------------- | ------------------------------------------------------------ |
| **YOLOv1-v5** (2016-2020)   | 是（从 v2 开始） | 使用 k-means 聚类生成 anchor，提高多尺度检测效率，但 anchor 需手动/自动优化，泛化性差。 |
| **YOLOv6-v7** (2022)        | 是（部分）       | 仍依赖 anchor，但引入更多自动化（如 auto-anchor）。          |
| **YOLOv8** (2023)           | **否**           | 首次全面 anchor-free，直接预测对象中心点 + 宽高偏移，简化模型、提升精度和速度。 |
| **YOLOv9-v10** (2024)       | **否**           | 继承 v8 设计，进一步优化（如 GELAN 骨干），anchor-free 已成为主流。 |
| **YOLOv11** (2024 年 10 月) | **否**           | 强化 anchor-free，支持更灵活的边界框生成，尤其对小/不规则物体友好。 |



# 转向无锚框的原因

-   简化，不需要预定义锚框，不用调锚框超参
-   锚框容易转向训练数据
-   速度快，mAP值高

**YOLO 现在预测“对象中心 + 尺寸”，而非“anchor 偏移”。**

# 先前使用先验锚框的原因

-   为了解决在同一个位置的多个物品需要检测的问题，不同的物品尺寸不同
-   一个网格必须有多个“起点”（先验框）才能负责不同尺寸的目标
-   **anchor 把“从零开始回归任意框”这个超难任务，变成了“在几个比较靠谱的模板上做小偏移回归”这个简单得多的任务。**

| 问题                               | 没有 anchor 的后果                    | Anchor 解决的方式                                |
| ---------------------------------- | ------------------------------------- | ------------------------------------------------ |
| 一个网格要预测多种尺度的目标       | 回归难度极大，梯度相互干扰            | 给每个网格预设 3~9 个不同尺寸/比例的“模板框”     |
| 长宽比变化大（如人 vs 汽车 vs 猫） | 模型很难直接学到极端比例              | 用 k-means 聚类出真实数据集常见的几种比例        |
| 训练初期框的位置完全随机           | 前几百个 epoch 几乎全是负样本，学不动 | 先验框已经比较靠近真实框，IoU 更高，正样本更多   |
| 小物体检测极差                     | 小物体占像素少，回归误差敏感          | 专门给浅层特征图配小 anchor（比如 16×16、32×32） |

## 弃用锚框的原因

| 技术进步                                             | 让模型摆脱 anchor 的关键作用                             |
| ---------------------------------------------------- | -------------------------------------------------------- |
| 更深的 backbone + FPN                                | 不同尺度的特征图天然负责不同大小的物体，无需 anchor 分工 |
| 更好的回归损失（GIoU、DIoU、CIoU）                   | 让直接回归大框也不再那么难                               |
| 更强的正样本分配策略（SimOTA、ATSS、AutoAssign）     | 动态决定哪个网格负责哪个物体，不再依赖固定 anchor        |
| 更强的标签分配（Distribution Focal Loss、VARIFOCAL） | 把“中心点+宽高分布”建模得更科学                          |
| 训练技巧（EMA、更大分辨率、更多数据）                | 模型容量和优化足够强大，直接回归也收敛了                 |

“直接预测中心点 + 宽高”反而变得比“基于 anchor 的偏移回归”更简单、更准、泛化更好。

## 还使用锚框的网络

| 场景                          | 最常见的anchor-based模型             | 原因                                                         |
| ----------------------------- | ------------------------------------ | ------------------------------------------------------------ |
| 学术论文（尤其两阶段）        | Faster RCNN、Cascade RCNN、Mask RCNN | 审稿人最熟悉，baseline最稳                                   |
| 实例分割 / 关键点检测         | Mask RCNN、HTC、Detectron2全系列     | 几乎全部基于anchor-based RPN                                 |
| 国内很多企业老项目            | YOLOv5、YOLOv7、PP-YOLOE             | 已经调好、部署稳定、不敢轻易升级                             |
| 嵌入式/边缘设备老代码         | SSD、YOLOv5s、YOLOv7-tiny            | 历史包袱，代码改动成本高                                     |
| 需要极致精度（不在乎0.5~1AP） | PP-YOLOE-L/X（百度系）               | 在同等速度下，PP-YOLOE系列目前仍比YOLOv11略高0.5~1.5 AP（靠anchor+更多trick） |
