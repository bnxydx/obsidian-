# TDNN

（时间延迟神经网络，Time Delay Neural Network）

核心思想：在每一层对时间维度做“延迟”拼接，即当前时刻的输入会拼接前后若干帧的特征，形成更宽的时间上下文感受野。

## spec（频谱图，Spectrogram）

最广义的语音时频表示，通常指短时傅里叶变换（STFT）后的幅度谱或功率谱：

在深度学习时代，很多模型（比如Wav2Vec2、ResNet类说话人识别、语音识别端到端模型）直接把原始波形转成spec（通常80～128维mel滤波器组的log-mel spec）作为输入。

-   与MFCC区别：spec保留了更完整的频谱信息（相位除外），维度更高，适合CNN或Transformer处理。

## MFCC梅尔频率倒谱系数

经典的传统语音特征



# SPEC 和 MFCC

是的，**spec** 和 **MFCC** 都属于音频的声学特征（Acoustic Features），但它们在表示方式、信息的完整程度和当前使用场景上有明显区别。可以把它们理解为“两代”特征：

| 项目                   | spec（频谱图）                                               | MFCC（梅尔频率倒谱系数）                   |
| ---------------------- | ------------------------------------------------------------ | ------------------------------------------ |
| 是否是音频特征         | 是                                                           | 是                                         |
| 原始形式               | 时频谱（通常是 log-mel spectrogram 或 log-power spectrogram） | 倒谱系数（cepstral coefficients）          |
| 信息保留程度           | 高（几乎保留所有频率细节，只有相位被丢弃）                   | 低（经过DCT截断，只保留前13～20维左右）    |
| 维度                   | 通常 80～256 维（每帧）                                      | 经典是 13维 + Δ + ΔΔ = 39维                |
| 是否可逆（能重构波形） | 配合Griffin-Lim或神经网络声码器基本可逆                      | 几乎不可逆（信息丢失太多）                 |
| 人耳听觉模拟           | 有（Mel滤波器组）                                            | 更强（Mel + log + DCT模拟人耳倒谱特性）    |
| 主要适配模型           | CNN、ResNet、Transformer、Wav2Vec、HuBERT、ECAPA-TDNN 等深度模型 | 传统 GMM-UBM、i-vector、早期TDNN           |
| 当前主流程度           | 绝对主流（2020年后99%的SOTA模型都用log-mel spec或FBank）     | 基本被淘汰，仅在极轻量或传统系统里偶尔出现 |

一句话总结：
- **MFCC** 是 2000～2015 年左右的“经典手工特征”，为当时统计模型（GMM）量身定做；
- **spec**（尤其是 log-mel spectrogram / FBank）是 2018 年之后深度学习时代的“新标准特征”，几乎所有现代说话人识别、语音识别、声纹模型都直接吃它。

所以现在如果你看到论文或开源代码里写输入特征，大概率是 80-dim log-mel spectrogram，而不是39-dim MFCC了。MFCC 已经算是“上一代”特征了。





# 波形如何变成维度

语音从“波形”变成“维度”（也就是每一帧变成一个向量）要经过下面这几步，每一步都在增加或改变“维度”的含义。  
我用最常用的 **80维 log-mel spectrogram**（当前工业界/学术界绝对主流）为例，完整走一遍整个流程，你就知道“维度”是怎么一步步产生的：

| 步骤 | 操作                           | 输出是什么                                            | 维度怎么来的                                                 |
| ---- | ------------------------------ | ----------------------------------------------------- | ------------------------------------------------------------ |
| 1    | 原始波形（raw waveform）       | 1维时间序列，比如 16kHz 采样率，3秒就是 48000 个点    | 1维（只是时间）                                              |
| 2    | 预加重 + 分帧 + 加窗           | 把波形切成很多短时帧（frame），通常帧长25ms，帧移10ms | 每帧约 400 个采样点（16kHz × 0.025s）                        |
| 3    | 短时傅里叶变换（STFT）         | 每帧得到一个复数频谱 → 取幅度 → 得到幅度谱            | 频谱分辨率通常是 512 点（FFT点数）→ 257维实数（对称只取前半） |
| 4    | Mel滤波器组（Mel filter bank） | 用 80 个三角形Mel滤波器去“压一压”这257维频谱          | 257维 → 80维（每个滤波器输出一个能量值）                     |
| 5    | 取对数（log）                  | log(能量) → log-mel spectrogram                       | 还是 80维，只是数值变成了对数尺度                            |
| 6    | （可选）添加 Δ + ΔΔ            | 再拼接一阶、二阶差分                                  | 80 → 240维（很少用了，现在基本只用80维）                     |

最终结果：
- 每一帧语音（25ms）都被表示成一个 **80维的向量**（80-dim log-mel feature）
- 3秒语音 ≈ 300 帧 → 得到一个 **300 × 80** 的二维矩阵
- 这个矩阵就是深度神经网络（CNN、TDNN、ECAPA-TDNN、Transformer等）的输入

用图示意一下整个过程：

```
原始波形 (1维)
   ↓ 分帧加窗 (25ms/帧)
每帧 400 个采样点
   ↓ STFT
每帧 257 维幅度谱
   ↓ 80个Mel三角滤波器
每帧 80 维 Mel 能量
   ↓ 取对数
每帧 80 维 log-mel 向量  ←←←← 这就是我们说的“80维特征”
   ↓ 按时间堆叠
T 帧 × 80 维 的特征矩阵 → 输入神经网络
```

### 常见维度对照表（2025年真实使用情况）

| 特征类型                 | 每帧维度 | 备注                                |
| ------------------------ | -------- | ----------------------------------- |
| 经典MFCC                 | 39       | 13静态 + 13Δ + 13ΔΔ（2015年前主流） |
| FBank / log-mel spec     | 80       | 目前90%以上的模型使用               |
| 高分辨率log-mel spec     | 128      | 某些大模型（WavLM、HuBERT）会用     |
| 原始spectrogram（非mel） | 257/129  | 线性频谱，部分自监督模型会用        |

总结一句话：
语音是通过“分帧 → 傅里叶变换 → Mel滤波器组 → 取对数”这一系列操作，把连续的波形信号变成了**每一帧一个固定维度的向量**，这个向量维度就是我们常说的“40维、80维、128维特征”。  
80维 log-mel spectrogram 就是当前“事实上的行业标准维度”。
